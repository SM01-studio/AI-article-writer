## Phase 0: 初始化与需求收集

### 触发条件
用户调用 `/ai-article-writer [主题]` 或描述写作需求

### 执行步骤

#### Step 0.1: 解析用户输入
```
提取信息:
- 文章主题（必需）
- 目标读者（可选）
- 参考资料链接（可选）
- 特殊要求（可选）
- 截止时间（可选）
```

#### Step 0.2: 询问文章规格
使用 AskUserQuestion 工具:

```
问题: 请选择文章长度
选项:
- 短篇（1500-3000字）- 快速科普，适合碎片阅读
- 中篇（3000-5000字）- 全面介绍，标准深度
- 长篇（5000-10000字）- 深度剖析，详尽解读
```

#### Step 0.3: 确认目标读者（如用户未提供）
```
问题: 目标读者是谁？
选项:
- 技术从业者（有技术背景，可以使用专业术语）
- 产品/业务人员（了解概念即可，避免代码细节）
- 大众读者（零基础，需要大量类比和通俗解释）
- 混合受众（多层次，兼顾深度和可读性）
```

#### Step 0.4: 输出调研计划
```markdown
## 调研计划

### 将要调研的关键问题
1. [核心概念1]是什么？
2. [核心概念2]如何工作？
3. 最新发展趋势是什么？
4. 有哪些实际应用案例？

### 预估信息源类型
- 官方文档/技术博客
- 学术论文/研究报告
- 新闻报道/行业分析
- 开源项目/GitHub

### 预计调研时间
约 [X] 分钟
```

## Phase 1: 深度调研

### 目标
全面收集信息，建立知识基础

### 前置阅读
执行前阅读 [research-guide.md](research-guide.md)

### 执行步骤

#### Step 1.1: 核心概念搜索（WebSearch）
使用 WebSearch 搜索官方文档、学术论文:
```
搜索词示例:
- "[主题] 是什么"
- "[主题] 原理"
- "[主题] vs [竞品]"
- "[主题] 最新进展 2026"
- "[主题] official documentation"
- "[主题] research paper"
```

#### Step 1.2: 微信公众号搜索（MCP）
使用 mcp__weixin__weixin_search 搜索中文深度分析:
```
搜索词示例:
- "[主题] 解析"
- "[主题] 原理详解"
- "[主题] 应用案例"
- "[主题] 趋势"

获取文章内容:
- 使用 mcp__weixin__get_weixin_article_content(real_url, referer) 获取正文
```

#### Step 1.3: 小红书趋势搜索（MCP）
使用 mcp__xiaohongshu__search_notes 搜索用户体验和案例:
```
搜索词示例:
- "[主题] 体验"
- "[主题] 教程"
- "[主题] 案例"

获取笔记内容:
- 使用 mcp__xiaohongshu__get_note_content(url) 获取详情
- 使用 mcp__xiaohongshu__get_note_comments(url) 获取评论
```

#### Step 1.4: 深度阅读
使用 WebFetch / mcp__web_reader__webReader 访问高质量信息源:
```
访问顺序:
1. 官方文档/技术规范
2. 权威技术博客
3. 学术论文/研究报告
4. 行业新闻/评论
```

#### Step 1.5: 信息整理
```markdown
## 信息分类

### 核心概念
- [概念]: [解释] (来源: [URL])

### 技术细节
- [细节]: [说明] (来源: [URL])

### 案例数据
- [案例]: [描述] (来源: [URL])

### 最新动态
- [动态]: [描述] (来源: [URL])

### 争议观点
- [观点A]: [描述]
- [观点B]: [描述]
```

#### Step 1.6: 生成调研报告
```markdown
# 调研报告：[主题]

## 核心概念
[概念解释，每个概念1-2段]

## 技术原理
[工作原理说明，适合目标读者理解层次]

## 应用场景
[实际应用案例，包含具体数据]

## 发展趋势
[最新动态和未来展望]

## 争议与挑战
[不同观点和面临的挑战]

## 信息缺口
[需要补充调研的方向]

## 参考来源
1. [标题] - [URL] - [可信度评估]
2. ...
```

#### Step 1.7: 🔴 确认节点1
```
使用 AskUserQuestion:
问题: 调研报告已完成，是否继续生成大纲？
选项:
- 继续（满意）
- 需要补充调研（请说明方向）
- 重新调研（请说明问题）
```

## Phase 2: 大纲设计

### 目标
构建清晰的文章骨架

### 前置阅读
执行前阅读 [writing-standards.md](writing-standards.md)

### 执行步骤

#### Step 2.1: 确定文章结构

**短篇结构（1500-3000字）**:
```
引言（200字）
├── 钩子
├── 背景
└── 预告

核心段落1（600字）
核心段落2（600字）
核心段落3（400字）

结语（200字）
```

**中篇结构（3000-5000字）**:
```
引言（300字）
├── 钩子
├── 背景
└── 预告

第一章：是什么（800字）
第二章：为什么（800字）
第三章：怎么做（800字）
第四章：案例（600字）

结语（300字）
```

**长篇结构（5000-10000字）**:
```
引言（500字）
├── 钩子
├── 背景
├── 重要性
└── 预告

第一部分：基础篇（2000字）
├── 第一章：概念
├── 第二章：原理
└── 第三章：发展历程

第二部分：进阶篇（2500字）
├── 第四章：技术细节
├── 第五章：应用场景
└── 第六章：案例分析

第三部分：展望篇（1500字）
├── 第七章：趋势
└── 第八章：挑战

结语（500字）
```

#### Step 2.2: 设计章节内容
为每个章节确定:
- 核心论点（1句话）
- 支撑案例（1-2个）
- 数据支撑（如适用）
- 过渡逻辑

#### Step 2.3: 规划配图位置
```
配图规划:
- 封面图: [主题可视化描述]
- 章节1图: [概念可视化描述]
- 章节2图: [原理可视化描述]
- ...
```

#### Step 2.4: 生成大纲
```markdown
# 文章大纲：[标题]

## 元信息
- 目标长度: [短/中/长]
- 预计字数: [X]字
- 目标读者: [类型]
- 写作风格: [描述]

## 封面配图
- 视觉描述: [...]

## 引言（约[X]字）
- 钩子: [吸引读者的开场]
- 背景: [为什么这个话题重要]
- 预告: [文章将覆盖什么]

## 第一章：[标题]（约[X]字）
- 核心论点: [...]
- 支撑内容: [...]
- 配图描述: [...]

## 第二章：[标题]（约[X]字）
...

## 结语（约[X]字）
- 核心回顾
- 未来展望/行动建议

## 统计
- 总章节: [X]
- 预计配图: [X]张
- 预计字数: [X]字
```

#### Step 2.5: 🔴 确认节点2
```
使用 AskUserQuestion:
问题: 文章大纲已生成，是否满意？
选项:
- 确认，开始写作
- 需要调整结构（请说明）
- 需要调整重点（请说明）
```

## Phase 3: 内容写作

### 目标
按大纲撰写高质量内容

### 前置阅读
执行前阅读 [writing-standards.md](writing-standards.md)

### 执行步骤

#### Step 3.1: 逐章节写作
```
写作顺序:
1. 引言（设定基调）
2. 核心章节（按大纲顺序）
3. 结语（升华总结）
```

#### Step 3.2: 写作检查清单
每写完一个章节，检查:
- [ ] 是否符合大纲规划
- [ ] 语言是否适配目标读者
- [ ] 是否有具体案例支撑
- [ ] 段落过渡是否流畅
- [ ] 配图位置是否标注

#### Step 3.3: 生成初稿
```markdown
# [文章标题]

[引言内容...]

![封面图](images/cover.png)

## 第一章：[标题]

[章节内容...]

*配图位置：[配图描述]*

![章节1图](images/chapter-1.png)

## 第二章：[标题]
...

## 结语

[结语内容...]
```

#### Step 3.4: 🔴 确认节点3
```
使用 AskUserQuestion:
问题: 初稿已完成，请审核内容质量
选项:
- 满意，继续配图排版
- 需要修改内容（请说明具体修改意见）
- 需要调整风格（请说明期望）
```

## Phase 4: 配图生成

### 目标
使用Gemini API生成高质量配图

### 核心原则
⚠️ **所有配图风格必须统一**

### 前置阅读
执行前阅读 [image-generation.md](image-generation.md)

### 执行步骤

#### Step 4.1: 确定统一风格
```
风格维度:
1. 色调: 冷色调（蓝/紫）/ 暖色调（橙/红） / 中性（黑白灰）
2. 风格: 科技感 / 简约 / 写实 / 插画 / 抽象
3. 元素: 几何图形 / 线条 / 粒子 / 渐变
4. 氛围: 专业 / 活泼 / 神秘 / 未来感

确定后记录风格关键词，所有配图复用:
风格关键词 = "[色调], [风格], [元素], [氛围], high quality"
```

#### Step 4.2: 生成配图Prompt
```
封面prompt = "[主题核心概念], [风格关键词]"
章节1 prompt = "[章节1核心概念], [风格关键词]"
章节2 prompt = "[章节2核心概念], [风格关键词]"
...
```

#### Step 4.3: 调用生成脚本
```bash
# 封面图
python scripts/generate_image.py \
  --prompt "[主题核心概念]" \
  --style "[风格关键词]" \
  --output "output/[文章标题]/images/cover.png"

# 章节图（循环生成）
python scripts/generate_image.py \
  --prompt "[章节核心概念]" \
  --style "[风格关键词]" \
  --output "output/[文章标题]/images/chapter-1.png"
```

#### Step 4.4: 风格一致性检查
```
检查项:
- [ ] 所有图片色调一致
- [ ] 所有图片风格一致
- [ ] 图片与内容主题契合
- [ ] 图片质量符合要求

如发现偏差:
1. 使用相同的风格关键词重新生成
2. 调整风格描述词的权重
```

## Phase 5: 排版输出

### 目标
生成多格式富文本输出

### 前置阅读
执行前阅读 [output-formats.md](output-formats.md)

### 执行步骤

#### Step 5.1: 询问输出格式
```
使用 AskUserQuestion:
问题: 请选择输出格式
选项:
- 微信公众号格式（带内联样式）
- HTML富文本（完整网页）
- 两种格式都输出
```

#### Step 5.2: 应用排版模板
使用 `assets/templates/` 目录中的模板:
- `wechat.html` - 微信公众号模板
- `article.html` - HTML富文本模板

#### Step 5.3: 插入配图
```
图片路径:
- 微信格式: 使用相对路径或占位符（发布时替换）
- HTML格式: 使用相对路径 images/xxx.png
```

#### Step 5.4: 🔴 确认节点4
```
使用 AskUserQuestion:
问题: 排版已完成，请确认最终输出
选项:
- 确认，输出最终文件
- 需要调整样式（请说明）
```

#### Step 5.5: 输出文件
```
output/[文章标题]/
├── content.md          # Markdown原文
├── wechat.html         # 微信公众号格式
├── article.html        # HTML富文本格式
└── images/
    ├── cover.png
    ├── chapter-1.png
    └── ...
```

## 异常处理指南

### 调研信息不足
```
情况: 搜索结果少，信息源质量低
处理:
1. 扩大搜索范围（中英文、不同搜索引擎）
2. 告知用户信息缺口，询问是否有补充资料
3. 基于有限信息写作，标注"需要进一步验证"
```

### 用户反复修改
```
情况: 用户多次要求修改同一内容
处理:
1. 询问具体不满意的地方
2. 提供2-3个不同版本供选择
3. 建议分阶段确认，避免大范围返工
```

### 配图生成失败
```
情况: Gemini API调用失败或生成质量差
处理:
1. 检查API配置是否正确
2. 简化prompt，减少复杂描述
3. 尝试不同的风格关键词
4. 如持续失败，告知用户稍后手动添加配图
```

### 输出格式问题
```
情况: 排版在目标平台显示异常
处理:
1. 检查CSS样式是否兼容
2. 微信格式: 使用内联样式，避免外部CSS
3. HTML格式: 检查响应式设计
```
