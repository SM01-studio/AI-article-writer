## 调研原则

### 1. 多源验证
- 每个关键信息至少需要2-3个独立来源验证
- 优先使用一手资料（官方文档、论文原文）
- 避免仅依赖二手报道

### 2. 时效性优先
- AI领域发展迅速，优先使用近6个月的资料
- 对于技术细节，优先使用最新版本文档
- 标注信息的时效性（如"截至2024年Q3"）

### 3. 平衡视角
- 收集正面和负面观点
- 包含不同利益相关者的声音
- 呈现争议时保持中立

### 4. 读者适配
- 根据目标读者调整调研深度
- 技术读者：深入技术细节
- 大众读者：聚焦概念和应用

## MCP搜索工具

> 本系统集成了两个 MCP 搜索工具，用于深度调研阶段的信息收集

### 微信公众号搜索 MCP

用于搜索和获取微信公众号文章内容，适合获取中文深度分析文章。

**可用工具：**

| 工具 | 功能 | 使用场景 |
|------|------|----------|
| `weixin_search` | 搜索微信公众号文章 | 查找相关主题的公众号文章 |
| `get_weixin_article_content` | 获取文章正文内容 | 提取文章详细内容用于分析 |

**使用示例：**
```
# 搜索文章
调用 weixin_search("大语言模型原理")

# 获取文章内容
调用 get_weixin_article_content(real_url, referer)
```

**注意事项：**
- 搜索结果包含标题、摘要、公众号名称和链接
- 获取内容时需要使用搜索返回的 real_url 和 referer

### 小红书搜索 MCP

用于搜索小红书笔记内容，适合获取用户真实体验、应用案例和趋势观察。

**可用工具：**

| 工具 | 功能 | 使用场景 |
|------|------|----------|
| `search_notes` | 搜索小红书笔记 | 查找相关主题的笔记 |
| `get_note_content` | 获取笔记详细内容 | 提取笔记完整内容 |
| `get_note_comments` | 获取笔记评论 | 了解用户反馈和讨论 |

**使用示例：**
```
# 搜索笔记
调用 search_notes(keywords="AI写作工具", limit=10)

# 获取笔记内容
调用 get_note_content(url="https://www.xiaohongshu.com/explore/xxxx")

# 获取评论
调用 get_note_comments(url="https://www.xiaohongshu.com/explore/xxxx")
```

**注意事项：**
- 首次使用需要扫码登录小红书
- 建议每天评论相关操作不超过30条
- 运行时会打开真实浏览器窗口

### 调研中的使用策略

```
Phase 1 深度调研流程:

1. 初始搜索（WebSearch）
   ├── 搜索官方文档、学术论文
   └── 获取技术原理和权威信息

2. 微信公众号补充（weixin_search MCP）
   ├── 搜索中文深度分析文章
   ├── 获取行业专家观点
   └── 收集本土化案例

3. 小红书趋势观察（xiaohongshu MCP）
   ├── 搜索用户真实体验
   ├── 了解大众认知和应用
   └── 发现热点话题和争议

4. 信息整合
   ├── 交叉验证关键信息
   ├── 评估信息可信度
   └── 生成调研报告
```

### 信息源优先级

| 优先级 | 来源 | MCP工具 | 可信度 |
|--------|------|---------|--------|
| 1 | 官方文档、学术论文 | WebSearch | 最高 |
| 2 | 微信公众号深度文章 | weixin_search | 高 |
| 3 | 技术博客、行业报告 | WebSearch | 中高 |
| 4 | 小红书用户分享 | xiaohongshu | 中（需验证） |

## 可信度评估标准

### 评估维度

```
可信度评分 (1-5):
├── 来源权威性 (1-5)
│   ├── 官方/学术机构 = 5
│   ├── 知名媒体/专家 = 4
│   ├── 行业博客 = 3
│   ├── 个人博客 = 2
│   └── 匿名来源 = 1
│
├── 信息时效性 (1-5)
│   ├── 1个月内 = 5
│   ├── 3个月内 = 4
│   ├── 6个月内 = 3
│   ├── 1年内 = 2
│   └── 1年以上 = 1
│
├── 证据充分性 (1-5)
│   ├── 数据+引用完整 = 5
│   ├── 有数据支撑 = 4
│   ├── 逻辑清晰 = 3
│   ├── 观点为主 = 2
│   └── 无证据 = 1
│
└── 独立性 (1-5)
    ├── 独立研究 = 5
    ├── 有利益披露 = 4
    ├── 可能有利冲 = 2
    └── 明显商业推广 = 1
```

### 评估示例
```
来源: OpenAI官方博客 - GPT-4技术报告
评分:
- 来源权威性: 5 (官方)
- 信息时效性: 4 (发布于6个月内)
- 证据充分性: 5 (包含详细数据)
- 独立性: 3 (公司自评)
综合评分: 4.25/5

结论: 高可信度，技术细节可信，但需注意可能的商业倾向
```

## AI主题调研重点

### 大语言模型 (LLM)
```
必调研项:
├── 核心架构（Transformer变体）
├── 参数规模和性能指标
├── 训练数据来源
├── 上下文窗口大小
├── 推理能力（CoT, 规划等）
├── 局限性和已知问题
└── 许可和使用限制

推荐搜索词:
- "[模型名] technical report"
- "[模型名] architecture"
- "[模型名] vs [竞品] benchmark"
```

### AI应用场景
```
必调研项:
├── 具体应用案例
├── 效果数据（准确率、效率提升等）
├── 实施难度和成本
├── 用户反馈
├── 潜在风险和挑战
└── 最佳实践

推荐搜索词:
- "[应用场景] AI implementation case study"
- "[应用场景] AI ROI"
- "[应用场景] AI challenges"
```

### AI趋势和未来
```
必调研项:
├── 最新研究突破
├── 行业投资动向
├── 专家预测和观点
├── 潜在风险和伦理问题
├── 政策法规动态
└── 人才和市场趋势

推荐搜索词:
- "AI trends 2024"
- "AI investment report"
- "AI ethics challenges"
```
