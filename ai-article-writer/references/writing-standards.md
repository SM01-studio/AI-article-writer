## 写作原则

### 1. 准确性优先
- 技术术语使用准确
- 数据来源可追溯
- 避免过度简化导致误解
- 不确定时标注"据公开信息"

### 2. 可读性兼顾
- 复杂概念用类比解释
- 长句拆分，每句不超过30字
- 段落控制在3-5句
- 使用小标题提高可读性

### 3. 客观中立
- 呈现多方观点
- 避免绝对化表述
- 承认局限性和不确定性
- 区分事实和观点

### 4. 价值导向
- 解决读者关心的问题
- 提供可操作的信息
- 避免纯粹的技术炫耀
- 关注实际应用价值

## 读者适配指南

### 技术从业者
```
语言特点:
- 可以使用专业术语
- 适当保留英文原文
- 可以包含技术细节
- 可以有代码示例

内容侧重:
- 技术原理和实现
- 性能指标对比
- 最佳实践
- 踩坑经验

示例:
"Transformer的自注意力机制通过Q、K、V三个矩阵计算..."
```

### 产品/业务人员
```
语言特点:
- 减少技术术语
- 聚焦应用价值
- 使用商业案例
- 避免代码细节

内容侧重:
- 能做什么（能力边界）
- 应用场景
- 成本效益
- 实施难度

示例:
"大语言模型可以理解上下文，就像一个读过大量书籍的助手..."
```

### 大众读者
```
语言特点:
- 完全避免术语或详细解释
- 大量使用类比和比喻
- 联系日常生活
- 故事化表达

内容侧重:
- 是什么（通俗易懂）
- 对我有什么影响
- 未来会怎样
- 有趣的案例

示例:
"想象一下，你有一个超级聪明的机器人朋友，它读过互联网上所有的书..."
```

### 混合受众
```
策略:
- 分层写作：先通俗解释，再深入技术
- 使用"技术角"或"深入理解"板块
- 核心内容面向主要读者群体
- 提供不同阅读路径

示例:
"大语言模型（通俗来说就是...）是一种...

【技术角】从技术角度看，LLM基于Transformer架构..."
```

## 文章结构规范

### 引言结构
```markdown
## [引人注目的标题]

[钩子 - 1-2句话吸引读者]

[背景 - 为什么这个话题重要]

[预告 - 文章将覆盖什么，读者能获得什么]

---
```

**钩子示例**:
- 疑问式："你有没有想过，ChatGPT是怎么理解你说的话的？"
- 数据式："2024年，AI市场规模突破了XXX亿美元..."
- 故事式："去年，一家小公司用AI做到了..."
- 对比式："三年前还需要XX，现在只需要..."

### 章节结构
```markdown
## 章节标题

[核心观点 - 1-2句话]

[详细解释 - 2-3段]

[案例/数据 - 具体支撑]

[小结/过渡 - 承上启下]

---
```

### 结语结构
```markdown
## 结语

[核心回顾 - 3-5个要点]

[个人观点 - 可选，需标注]

[未来展望 - 趋势预测]

[行动建议 - 读者可以做什么]
```

## 语言风格指南

### 句式规范

**好句式**:
```
✓ 短句为主，每句15-25字
✓ 主谓宾结构清晰
✓ 适当使用设问句
✓ 列表呈现多项信息
```

**避免**:
```
✗ 长句超过40字
✗ 多重嵌套从句
✗ 被动语态过多
✗ 形容词堆砌
```

### 类比使用

**好的类比**:
```
✓ 贴切：类比对象与主题有相似的核心特征
✓ 熟悉：读者能立即理解
✓ 简单：不过度复杂化

示例:
- 训练数据 → 模型的"教材"
- 参数 → 模型的"神经元连接"
- 提示词 → 给模型的"指令"
```

**避免**:
```
✗ 过度延伸类比
✗ 使用读者不熟悉的类比
✗ 类比本身比原概念还复杂
```

### 专业术语处理

**首次出现时**:
```markdown
大语言模型（Large Language Model，简称LLM）是...
```

**后续使用**:
```markdown
LLM在自然语言处理任务中表现优异...
```

**术语解释格式**:
```markdown
**[术语]**：[一句话解释]
[详细解释段落]
```

## 常见问题处理

### 如何处理技术细节？

```
原则：根据读者层次调整细节深度

大众读者：
- 跳过技术细节
- 用"黑盒"视角解释

技术读者：
- 保留技术细节
- 提供进一步阅读链接

混合受众：
- 核心内容简洁
- 技术细节放在"深入理解"板块
```

### 如何处理争议性话题？

```markdown
## [争议话题]

### 不同观点

**观点A**：[描述]
支持者认为：[理由]
代表人物/机构：[名称]

**观点B**：[描述]
支持者认为：[理由]
代表人物/机构：[名称]

### 分析
[客观分析各方论据]

### 结论
[如无法定论，承认不确定性]
```

### 如何处理不确定信息？

```markdown
正确示例：
- "据公开信息显示..." - 标注信息来源
- "截至2024年Q3..." - 标注时效性
- "有分析认为..." - 区分事实和观点
- "具体数据尚未公开" - 承认信息缺失

错误示例：
- "肯定是..." - 过于绝对
- "所有人认为..." - 以偏概全
- 不标注来源地引用数据
```

### 如何避免常见错误？

```
1. 不要把模型能力绝对化
   ✗ "ChatGPT可以理解一切"
   ✓ "ChatGPT在特定任务上表现出色"

2. 不要忽视局限性
   ✗ 只说优点
   ✓ 同时说明适用范围和局限

3. 不要过度简化
   ✗ "AI就是..."
   ✓ "AI包括多种技术，其中..."

4. 不要混淆概念
   ✗ 混用"机器学习"和"深度学习"
   ✓ 准确区分术语含义

5. 不要忽视伦理问题
   ✗ 只谈技术不谈风险
   ✓ 适当讨论潜在风险和伦理考量
```
